# Review 006 — Institutional compliance dashboard (UI)

**Branch: `task/006-ui`. Implementer: CC (Codex pass-1 read layer; CC pass-2 write flows). Reviewer: CC live + Codex read-only cross-pass.**

## Verdict: APPROVE — UI complete and verified live on testnet

> Process note: Codex implemented pass 1 (read layer + scorer `--json`) in the foreground
> successfully. Pass 2 (write flows) repeatedly **stalled when the harness auto-backgrounded
> the long Codex run** (0 bytes, 2 attempts), so CC implemented pass 2 directly. Cross-pass was
> preserved by having **Codex do an independent read-only review** of the finished UI.

## What was built
- **Pass 1 (Codex):** `lib/abis.ts`, `lib/chain.ts`, live read-only dashboard (`app/page.tsx`:
  header + Identity & Risk table + holdings), and a `--json` flag on `scorer/src/cli.ts`.
- **Pass 2 (CC):** `lib/server.ts` (server-only wallets + scorer runner) and 7 API routes
  (`score, attest, can-transfer, transfer, freeze, wrap, unwrap`) + interactive client panels
  (`components/Panels.tsx`) wired into the page; `ui/README.md`.

## Live verification (CC, against HashKey testnet)
- Production build clean; server renders the dashboard with real on-chain data.
- `/api/score` (dead) → 100 high + full breakdown. `/api/can-transfer`: Alice→Bob **allowed**,
  Alice→dead **false "recipient failed AML screen"**. `/api/transfer`: Alice→dead **blocked**
  (reason correct after fix), Alice→Bob **real tx hash**. `/api/freeze` Alice → tx (state restored).
  `/api/wrap` + `/api/unwrap` back-to-back both succeed.
- **Client bundle secret scan: CLEAN** — `.next/static` contains no private keys or key env names;
  `process.env.PRIVATE_KEY/ALICE_KEY` referenced only server-side.

## Fixes during review (CC)
- `lib/abis.ts`: added `transfer` (+ `approve`/`allowance`) — build type error otherwise.
- `transfer` route: rewrote `revertReason` to use viem `BaseError.walk` + `ContractFunctionRevertedError.reason` (was returning a stray "with the following reason:" fragment).
- All write routes: `waitForTransactionReceipt` before responding — fixes inter-request nonce
  collisions (wrap→unwrap, freeze→unfreeze) and gives fresh state on `router.refresh()`.

## Codex cross-pass (read-only): APPROVE, no P0/P1
- Confirmed no client-side key leakage; ABI args match contracts; transfer simulate-then-send and
  wrap/unwrap allowance flows correct.
- **P2 (applied):** receipt `status` wasn't checked — a mined-but-reverted tx could report success.
  CC added `receipt.status !== "success"` guards to all 5 write routes.

## Decision
Approved and merged to `master`. UI gap closed. Demo can now be recorded against the dashboard.
Remaining roadmap: optional wallet-connect, demo-script update to feature the UI (task 007 / C).
