# Task 006 ‚Äî Institutional compliance dashboard (UI)

**Owner (implement): Codex.  Reviewer (cross-pass): Claude Code.**
**Branch:** `task/006-ui`

## Goal

Close the biggest demo gap: a clean web dashboard ‚Äî a compliance officer's console ‚Äî that
reads the LIVE HashKey testnet deployment and drives every demo beat visually. This is the
single highest win-probability lever; it must look professional and demo cleanly on video.

## Design (decided by CC ‚Äî implement to this)

- **Stack:** Next.js (App Router) + TypeScript + Tailwind + `viem`. Location: `contracts-evm/ui/`.
  CC scaffolds the Next app + installs deps; you build pages/components/API routes.
- **Reads:** a viem public client reads live testnet (chainId 133, RPC `https://testnet.hsk.xyz`).
- **Writes:** Next.js **API routes that sign server-side** with keys from `.env.local`
  (gitignored): agent actions use `PRIVATE_KEY` (deployer holds AGENT/ISSUER/SCORER roles),
  the user-payment demo uses `ALICE_KEY`. No browser wallet needed (keeps the recorded demo
  reliable). Never expose a key to the client; keys only in API routes.
- **Config:** read all addresses from server env (`REGISTRY/AML/COMPLIANCE/TOKEN/MOCKHSP/
  WRAPPER/DEMO_ALICE/DEMO_BOB`). Provide `ui/.env.local.example`. A `lib/config.ts` (CC stub)
  exposes addresses + ABIs to the app.
- **Scorer integration:** the AML panel calls an API route that runs the scorer and returns
  JSON. To support this, ADD a `--json` flag to `scorer/src/cli.ts` that prints the
  `RiskResult` as JSON (no other output) ‚Äî the API route shells out
  (`tsx scorer/src/cli.ts score <addr> [--counterparty ‚Ä¶] --json`) and parses it. (This is the
  only allowed change outside `ui/`.)

## Sections (one cohesive dashboard; tabs or a single scroll)

1. **Header** ‚Äî chain name/id + the 6 contract addresses as explorer links.
2. **Identity & Risk** ‚Äî a table for Alice / Bob / the watchlisted `0x‚Ä¶dead` (+ an address
   lookup): KYC level, jurisdiction, frozen?, on-chain AML score + band. Live from chain.
3. **AML screening** ‚Äî input an address (+ optional counterparties) ‚Üí calls the scorer API ‚Üí
   shows score / band / per-feature breakdown ‚Üí an **"Anchor on-chain"** button ‚Üí `attestRisk`
   via API; the Identity table reflects the new on-chain score.
4. **Payment simulator** ‚Äî from / to / amount; show `canTransfer` result (ALLOWED / BLOCKED +
   reason) live; an **Execute** button sends the real tx via API. Quick buttons: Alice‚ÜíBob
   (allowed) and Alice‚Üí`dead` (blocked ‚Äî surface "recipient failed AML screen").
5. **Agent console** ‚Äî freeze / unfreeze an address; forced-transfer (recovery).
6. **Wrap / unwrap** ‚Äî wrap/unwrap MockHSP‚ÜîcHSP; show balances + the 1:1 backing.

## Acceptance criteria

- `pnpm --dir ui build` (Next production build) succeeds; `pnpm --dir ui lint` clean if eslint present.
- App runs (`pnpm --dir ui dev`) and every section renders against live testnet; the four
  write flows (attestRisk, payment execute, freeze, wrap/unwrap) work end-to-end via API routes.
- Payment simulator clearly shows Alice‚ÜíBob ALLOWED and Alice‚Üídead BLOCKED with the reason.
- No private key reaches the client bundle (keys only in `app/api/**` route handlers / server).
- Professional, legible styling (it's an institutional console). Add `ui/README.md`.
- `forge test` and scorer tests still green (you only add `--json` to the scorer CLI).
